Пошаговая инструкция по запуску

Для успешного локального запуска выполни шаги по порядку.

Шаг 1: Подготовка базы данных (PostgreSQL)

Открой pgAdmin или любой другой инструмент управления PostgreSQL.

Создай базу данных с именем task_db (без пробелов на конце).

Открой Query Tool для созданной базы task_db, скопируй код из файла database/init.sql и выполни его (нажми F5).

Скрипт создаст таблицу tasks со всеми необходимыми полями и ограничениями статусов (new, in_progress, done), а также добавит две тестовые задачи.

Шаг 2: Настройка и запуск бэкенда (Node.js)

В терминале VS Code перейди в папку бэкенда:

cd backend

Создай файл .env в папке backend/ и укажи в нём свои данные для подключения:

PORT=5000
DB*USER=postgres
DB_PASSWORD=твой*пароль_от_postgres # замени на свой актуальный пароль!
DB_HOST=localhost
DB_PORT=5432
DB_DATABASE=task_db

Установи зависимости:

npm install

Запусти сервер в режиме разработки:

npm run dev

Сервер запустится на порту 5000. В терминале должно отобразиться:
🚀 Сервер запущен на порту 5000
✅ База данных успешно подключена!

Шаг 3: Запуск фронтенда (React)

Открой новое окно терминала в VS Code и перейди в папку фронтенда из корня проекта:

cd frontend

Установи все необходимые npm-пакеты:

npm install

Запусти сервер разработки Vite:

npm run dev

Открой в браузере предоставленную ссылку (обычно это http://localhost:5173).

Теперь ты можешь полноценно использовать приложение: добавлять новые задачи, менять их статус в выпадающем списке и удалять.

Шаг 4: Запуск скрипта экспорта в CSV (Python)

Скрипт предназначен для выгрузки актуального списка задач в читаемый формат .csv, который корректно открывается в Microsoft Excel без проблем с кодировкой русского текста.

Открой терминал и перейди в папку со скриптами:

cd scripts

Убедись, что в коде файла export_tasks.py переменная DB_PASSWORD содержит твой пароль от PostgreSQL.

Установи драйвер подключения к БД (если не устанавливал ранее):

pip install psycopg2-binary

Запусти скрипт:

python export_tasks.py

После успешного выполнения в папке scripts/ появится файл tasks_export.csv.
